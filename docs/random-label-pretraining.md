# Random-Label Pre-Training Control: Is the Failure About Wrong Decisions or Extra Trees?

The XGBoost RLM→PAMPA transfer collapses to random-baseline performance (0.852 AUC-PR vs 0.855 baseline). The README argues this is because inherited RLM decision boundaries encode *wrong* task-specific associations that mislead the PAMPA model. But an alternative explanation is simpler: maybe *any* extra trees degrade the model, regardless of what they learned. If XGBoost's continue-boosting transfer is structurally harmful -- because the new trees must allocate capacity to correct old predictions rather than learning new patterns -- the content of the pre-training would be irrelevant.

This experiment distinguishes the two explanations.

## Design

Three conditions, all evaluated on PAMPA using the same 25-fold CV (5 replicates x 5 folds):

1. **Scratch** -- no pre-training. Baseline.
2. **RLM transfer** -- pre-train on real RLM labels (200 rounds), then continue boosting on PAMPA.
3. **Random-label transfer** -- pre-train on *shuffled* RLM labels (200 rounds, same molecules, same class balance, no real signal), then continue boosting on PAMPA. Five different random seeds for the label permutation, 25 folds each = 125 observations.

If random-label transfer collapses similarly to RLM transfer, the failure is structural (any extra trees are harmful). If random-label transfer performs substantially better than RLM transfer, the content of the inherited decisions matters -- *wrong* signal is worse than *no* signal.

## Interpretation

**Both explanations have merit, but the content of the inherited decisions is the dominant factor.**

Random-label transfer degrades PAMPA performance modestly relative to scratch: -0.016 AUC-PR (0.915→0.899) and -0.036 AUC-ROC (0.665→0.629). This confirms there *is* a structural cost to having extra trees in the ensemble -- the new PAMPA trees must work around 200 pre-existing trees that contribute noise to every prediction, and this costs some capacity.

But real RLM transfer is far worse: -0.063 AUC-PR (0.915→0.852) and -0.175 AUC-ROC (0.665→0.490). The gap between random-label transfer and real RLM transfer (0.047 AUC-PR, 0.139 AUC-ROC) is much larger than the gap between scratch and random-label transfer. The RLM trees don't just add noise -- they add *coherent, wrong signal*. They systematically push predictions in the wrong direction for PAMPA (e.g., encoding "nitrogen heterocycle → unstable" when PAMPA needs "nitrogen heterocycle → permeable or irrelevant"), and the new trees must fight against this organized misdirection rather than merely filtering out random noise.

The structural cost of extra trees is real but recoverable (random-label transfer stays well above the random baseline of 0.855 AUC-PR). The cost of inheriting *wrong* decision boundaries is catastrophic and irrecoverable (RLM transfer sits at the random baseline). This supports the README's framing: the failure mode is primarily about *wrong learned decisions*, not about the structural overhead of having more trees.

The per-seed breakdown shows the random-label result is stable across permutations (AUC-PR range: 0.896-0.908), confirming this is not an artifact of a particular label shuffle.
